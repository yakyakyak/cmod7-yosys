# DE10-Nano Port Plan

## Goal
Port the existing CMOD A7 design (LED blink + PWM + UART register interface) to the
Terasic DE10-Nano (Intel Cyclone V SoC 5CSEBA6U23I7) using Quartus Prime Lite.

## Board Facts
- **Device**: `5CSEBA6U23I7` (Cyclone V SoC, 110K LEs)
- **Clock**: `FPGA_CLK1_50` at `PIN_V11` — 50 MHz (vs 12 MHz on CMOD A7)
- **LEDs**: 8 green user LEDs (vs 2 on CMOD A7)
- **UART**: USB Mini-B goes to HPS ARM, NOT FPGA fabric. Use GPIO header pins
  with external USB-UART adapter for FPGA UART access.
- **Quartus install**: `~/altera_lite/25.1std/quartus/bin`
- **Cyclone V device support**: confirmed installed

## UART via GPIO_0 Header (external USB-UART adapter required)
Pick any two adjacent GPIO_0 pins. Suggested (first two user pins):
| Signal      | GPIO_0 Pin | FPGA Pin |
|-------------|------------|----------|
| uart_rx_in  | GPIO_0[0]  | PIN_V12  |
| uart_tx_out | GPIO_0[1]  | PIN_E8   |

## PWM Output
Pick any GPIO_0 pin. Suggested:
| Signal  | GPIO_0 Pin | FPGA Pin |
|---------|------------|----------|
| pwm_out | GPIO_0[2]  | PIN_W12  |

## Files to Create
All files go under `platforms/de10_nano/`:

### 1. `platforms/de10_nano/top.v`
Copy `platforms/cmod_a7/top.v` as starting point, then change:
- Module header comment: CMOD A7 → DE10-Nano, 12 MHz → 50 MHz
- Port `led` width: `[1:0]` → `[7:0]`
- `CLK_FREQ` parameter: `12_000_000` → `50_000_000` (baud rate divider)
- Port names can stay generic (`clk`, `led`, `uart_rxd_in`, `uart_txd_out`, `pwm_out`)
- LED assignments: `led[0]` and `led[1]` keep existing logic; `led[2:7]` tied low initially
- Counter width may stay 24-bit; blink rate will be 4x faster due to 50 MHz clock —
  consider widening to 26-bit to approximate same visible rate

### 2. `platforms/de10_nano/de10_nano.qsf`
Quartus Settings File (TCL format) with:
```tcl
set_global_assignment -name FAMILY "Cyclone V"
set_global_assignment -name DEVICE 5CSEBA6U23I7
set_global_assignment -name TOP_LEVEL_ENTITY top
set_global_assignment -name VERILOG_FILE ../../platforms/de10_nano/top.v
set_global_assignment -name VERILOG_FILE ../../src/pwm_generator.v
set_global_assignment -name VERILOG_FILE ../../src/reg_ctrl.v
set_global_assignment -name VERILOG_FILE ../../library/uart/uart_rx.v
set_global_assignment -name VERILOG_FILE ../../library/uart/uart_tx.v

# Clock
set_location_assignment PIN_V11  -to clk
set_instance_assignment -name IO_STANDARD "3.3-V LVTTL" -to clk
create_clock -name clk -period 20.000 [get_ports {clk}]

# LEDs
set_location_assignment PIN_W15  -to led[0]
set_location_assignment PIN_AA24 -to led[1]
set_location_assignment PIN_V16  -to led[2]
set_location_assignment PIN_V15  -to led[3]
set_location_assignment PIN_AF26 -to led[4]
set_location_assignment PIN_AE26 -to led[5]
set_location_assignment PIN_Y16  -to led[6]
set_location_assignment PIN_AA23 -to led[7]
# (repeat IO_STANDARD for each led pin)

# UART (GPIO_0 header, requires external USB-UART adapter)
set_location_assignment PIN_V12 -to uart_rxd_in
set_location_assignment PIN_E8  -to uart_txd_out

# PWM (GPIO_0 header)
set_location_assignment PIN_W12 -to pwm_out
```

### 3. `platforms/de10_nano/build.sh`
Quartus command-line build script:
```bash
#!/bin/bash
set -e
QUARTUS_BIN=~/altera_lite/25.1std/quartus/bin
PROJECT_DIR=$(dirname "$0")
PROJECT=de10_nano

export PATH=$QUARTUS_BIN:$PATH

cd "$PROJECT_DIR"
mkdir -p output_files

# Create project if not exists, then compile
quartus_sh --flow compile $PROJECT
```

The `.qpf` project file can be generated by quartus_sh or created manually.

## Build Output
- `platforms/de10_nano/output_files/de10_nano.sof` — program to SRAM (volatile)
- Program with: `openFPGALoader -b de10nano output_files/de10_nano.sof`
  or: `quartus_pgm -m jtag -o "p;output_files/de10_nano.sof"`

## Simulation
The existing `sim/` testbenches work for the CMOD A7 clock (12 MHz).
For DE10-Nano simulation, create `sim/de10_nano/` with updated testbenches:
- Copy `tb_top_quick.v` → adjust `CLK_PERIOD` to 20ns (50 MHz)
- Update DUT instantiation path to `platforms/de10_nano/top.v`
- Widen any counter checks to match 26-bit counter

Alternatively, pass `PLATFORM=de10_nano` to `simulate.sh` to select the right top.

## Key Differences from CMOD A7
| Aspect         | CMOD A7                    | DE10-Nano                         |
|----------------|----------------------------|-----------------------------------|
| Clock          | 12 MHz (L17)               | 50 MHz (PIN_V11)                  |
| CLK_FREQ param | 12_000_000                 | 50_000_000                        |
| LED count      | 2 (A17, C16)               | 8 (W15..AA23)                     |
| UART           | FTDI direct to FPGA pins   | GPIO header + external USB-UART   |
| Constraints    | `.xdc` (Xilinx)            | `.qsf` (Quartus TCL)              |
| Build tool     | OpenXC7 docker             | quartus_sh (native)               |
| Output file    | `.bit`                     | `.sof`                            |
| Programmer     | openFPGALoader -b cmoda7_35t | openFPGALoader -b de10nano      |
| Counter width  | 24-bit (ok at 12 MHz)      | 26-bit recommended (match rate)   |

## Step-by-Step Execution Order
1. Create `platforms/de10_nano/top.v` (copy + modify cmod_a7 top)
2. Create `platforms/de10_nano/de10_nano.qsf` (pin assignments)
3. Create `platforms/de10_nano/de10_nano.qpf` (Quartus project file)
4. Create `platforms/de10_nano/build.sh` (quartus_sh compile script)
5. Run `./platforms/de10_nano/build.sh` — check for errors
6. Fix any synthesis/fit errors
7. Program board: `openFPGALoader -b de10nano platforms/de10_nano/output_files/de10_nano.sof`
8. Verify LEDs blink at ~1 Hz
9. Connect USB-UART adapter to GPIO_0 pins, test UART register protocol
